### EiC-b — 결론 섹션 보강

**리뷰어 지적:**
결론이 초록과 내용이 겹치고, 연구의 강점·약점, 학문적 기여, 향후 연구 방향이 충분히 논의되지 않았다.

**대응:**
결론 섹션을 2단락에서 5단락으로 전면 재작성하였다. 각 단락의 역할은 다음과 같다:

1. **기여 요약** — 제안된 방법의 핵심 메커니즘(규칙 기반 대조 학습, 제로샷 영역 검출, 추론 시점 융합)이 기존 방법 대비 구체적으로 무엇을 개선했는지 정량적으로 서술. 단순 방법 설명인 초록과 차별화됨.
2. **강점 분석** — 규칙 기반 표현 학습의 해석 가능성, 제로샷 영역 검출의 무훈련(training-free) 특성, few-shot 설정에서의 효율성을 구체적으로 기술.
3. **한계 인정** — 수동 규칙 작성 의존성, 카운팅 유형 이상에서의 성능 약점, MVTec LOCO AD 외 벤치마크 검증 미비, 고정된 융합 가중치 λ의 한계.
4. **향후 연구** — 카테고리별 적응형 λ 선정, 자동화된 규칙 귀납(rule induction), 동영상 검사 영역으로의 확장.
5. **폭넓은 시사점** — 제조 검사를 넘어 인간 지식이 중요한 다른 도메인(의료 영상, 문서 이상 탐지 등)으로의 규칙 기반 패러다임 확장 가능성.

## 리뷰어 1 (R1)

### R1-1 — 점수 융합 모듈이 단순하고 복잡도·지연을 증가시킴

**리뷰어 지적:**
추가된 보완적 점수 융합(complementary score fusion)이 구조적으로 단순하고 관습적이다. 더 심각한 문제는 전체 모델의 복잡도와 추론 지연을 증가시켜 고효율을 요구하는 실제 산업 배포 환경에 적합하지 않다는 것이다.

**대응:**
점수 융합 모듈의 단순성은 의도적 설계 선택이며, 배포 우려에 직접 답하기 위해 정성적인 "무시할 수준"이라는 표현 대신 **실측 지연(latency)** 수치를 제시한다. 모든 측정은 RTX 3090 + ViT-B/16+ CLIP 백본 환경에서 수행하였다(다른 하드웨어/백본에서는 수치가 달라질 수 있다). 또한 23.5 ms 수치는 RRD 마스크가 이미 사전계산·캐시된 상태를 전제하며, 한 번도 본 적 없는 이미지를 처음 처리할 때는 아래 설명하는 ~678 ms 전처리가 1회 발생한다.

| 구성 요소 | 지연 | 실시간 경로 여부 | 학습 파라미터 |
|-----------|------|----------------|--------------|
| 논리 점수 단독 추론 (CLIP 이미지 forward 2회: 전역 + 사전계산 마스킹, + 텍스트 인코딩 + 융합) | **23.5 ± 2.2 ms / 이미지** | 예 | — |
| 점수 융합 연산 `s = (1−λ)·s_logic + λ·s_struct` | **~0.001 ms** | 예 | **0** |
| RRD 슬라이딩 윈도우 영역 검출 (320 패치) | ~678 ± 262 ms / 이미지 | **아니오 — 오프라인, 캐시됨** | 0 |
| 외부 구조적 탐지기 (WinCLIP / PromptAD), λ>0 일 때만 | 해당 탐지기 자체의 forward pass | 활성화 시에만 | 0 |

이 수치로부터 세 가지 사실이 직접 도출된다:

1. **융합 연산 자체는 사실상 무비용이다.** 융합 점수는 `s_fused = (1 − λ)·s_logic + λ·s_structural` 이라는 단일 스칼라 연산이며 **학습 가능한 파라미터가 0개**다. 실측 비용은 ~0.001 ms로, 23.5 ms 추론 예산의 약 4×10⁻⁵ 수준(10⁻⁴ 미만)이다. 따라서 제안된 모듈이 기본 논리 경로 위에 추가하는 "복잡도"는 실측상 의미 있게 측정되지 않는다.

2. **RRD의 678 ms는 추론마다 발생하는 비용이 아니라 오프라인에서 1회 분산(amortize)된다.** 마스킹 영역을 생성하는 슬라이딩 윈도우 영역 검출은 **이미지당 1회 전처리 단계로 실행되어 디스크에 캐시**된다(학습/추론 코드는 사전계산된 마스킹 텐서를 로드한다). 따라서 RRD는 실시간 추론 경로에 포함되지 않으며, 위의 23.5 ms 수치는 이미 사전계산된 마스크를 사용한 값이다(캐시되지 않은 새 이미지는 이 ~678 ms를 1회 부담한다). RRD가 질의마다 발생하는 런타임 비용이라는 오해를 막기 위해, 개정된 논문에 이 오프라인/온라인 분리를 명시적으로 기술하였다.

3. **융합 변형(variant)의 유일한 런타임 비용은 외부 탐지기 자체이며, 이는 선택 사항이다.** λ > 0 일 때 추가되는 런타임은 정확히 WinCLIP/PromptAD의 forward pass — 즉 그 외부 모델 고유의 비용(본 응답에서 별도 프로파일링하지는 않음)이지, 우리 융합이 새로 도입한 오버헤드가 아니다. 지연이 중요한 배포 환경에서는 **λ = 0** 을 설정하여 융합을 완전히 비활성화하면, 본 논문의 핵심인 논리 이상 탐지 능력을 그대로 유지한 채 **23.5 ms / 이미지**의 순수 논리 경로로 동작한다.

R3-4와 일관되게, 융합(λ > 0)은 구조적 단서가 존재할 때(MVTec-AD / VisA) 가장 유용하고 순수 논리 이상(LOCO)에서는 기여가 작다 — 이 경우 구조 점수가 논리 위반 신호와 대체로 직교(orthogonal)하므로 동일 가중 융합은 오히려 깨끗한 논리 점수를 희석할 수 있다. 따라서 논리 중심 배포에서 융합을 끄더라도 정확도 손실은 사실상 없을 것으로 예상되며(이는 R3-4 직교성 분석에 근거한 추론이며, λ = 0 대 λ > 0 의 정밀 비교는 개정된 논문에 보고함) 동시에 외부 탐지기의 런타임을 제거할 수 있다. 이 실측 지연 분해와 trade-off 논의를 개정된 논문에 추가하였으며, 카테고리별 적응형 λ(λ = 0 포함)를 자연스러운 확장 방향으로 명시하였다.

### R1-2 — 최신 SOTA 비교 부족

**리뷰어 지적:**
WinCLIP과의 비교만으로는 최신 zero-shot/few-shot 이상 탐지 방법들의 우세를 충분히 입증하기 어렵다. 더 많은 최신 SOTA 방법과의 비교가 필요하다.

**대응:**
실험 비교를 다음과 같이 확장하였다:

**1. WinCLIP 및 PromptAD 전 카테고리 비교**
MVTec LOCO AD 5개 카테고리 전체에서 shot=1 및 shot=5 두 조건 (총 10개 조건)을 평가하였다. LOCO 벤치마크에서 PromptAD가 WinCLIP을 10개 중 6개 조건에서 상회한다 — 이는 PromptAD가 LOCO에서 더 강력한 베이스라인임을 의미하며, 우리 방법이 더 어려운 베이스라인을 상대로도 경쟁력 있음을 보인다.

**2. 최신 CLIP/SigLIP/MLLM 기반 이상 탐지기와의 직접 비교 (신규)**
WinCLIP/PromptAD에 더해, 최근 발표된 탐지기 5종을 MVTec LOCO AD에서 각 방법이 의도한 설정 하에 직접 평가하였다(AnomalyCLIP/AdaCLIP는 기존 논문 수치, Anomaly-OV/APRIL-GAN/MuSc는 신규 추가). 모든 수치는 본 논문 전반에서 일관되게 사용하는 **이미지 수준 논리(logical-only) AUROC**(정상 vs 논리 이상)이다.

| 방법 | 설정 | 백본 | LOCO 논리 AUROC |
|------|------|------|----------------|
| Anomaly-OV | zero-shot (추론-MLLM 탐지 헤드) | SigLIP+LLaVA-OV | 0.534 |
| APRIL-GAN | few-shot (k=5), 크로스 데이터셋 | ViT-L/14-336 | 0.573 |
| AdaCLIP | zero-shot, 크로스 데이터셋 | ViT-L | 0.599 |
| AnomalyCLIP | zero-shot, 크로스 데이터셋 | ViT-L | 0.619 |
| MuSc | training-free transductive zero-shot | ViT-L/14-336 | 0.701 |
| **Ours** | **few-shot 1-shot / 5-shot** | **ViT-B/32** | **0.727 / 0.770** |

다섯 개 최신 탐지기 모두 우리의 few-shot 결과(1-shot 0.727 / 5-shot 0.770)에 미치지 못한다. 특히 가장 강력한 MuSc조차 **우리보다 2배 큰 ViT-L 백본**을 사용하고 **transductive 설정**(각 이미지를 전체 unlabeled test set 전부와 상호 비교하여 채점 — 우리의 per-image few-shot보다 유리한 조건)을 쓰면서도 논리 AUROC 0.701로 우리 1-shot에 미달한다. 추론-MLLM 탐지 헤드인 Anomaly-OV가 가장 낮으며(0.534), **모든 방법이 카운팅 카테고리(pushpins, screw bag)에서 붕괴**한다. 이는 논리 이상 격차를 메우는 핵심이 백본 규모나 transductive한 test set 접근이 아니라, **소수의 규칙 기반 정상 샘플(few-shot rule-grounded calibration)**임을 재확인한다.

**3. 일반 목적 VLM 비교 (R4-2 참조)**
Qwen2.5-VL-3B 및 7B를 직접 평가하였고, Q-72B 결과는 LAD-Reasoner 논문에서 인용하였다.

**4. AD 특화 추론 모델 비교 가능성**
- *LogiCode*: 주석 데이터만 공개, 추론 코드 및 체크포인트 미공개 → LOCO에서 모델 실행 불가, 인용으로만 다룸
- *IAD-R1*: 코드·체크포인트 공개되었으나 MVTec LOCO AD 평가 결과 미보고 → 저자 보고 수치 부재, 보완적 방향으로 논의
- *Anomaly-OV*: 탐지 헤드가 공개되어 **이제 직접 평가**하였다(zero-shot 논리 AUROC 0.534, 위 표 및 R4-2 참조)
- *OmniAD*: instruction-tuned 멀티모달 reasoner로 MMAD 벤치마크에서 평가됨(우리의 few-shot MVTec LOCO 프로토콜이 아님) → 동일 조건 비교 불가, 인용으로만 다룸

### R1-3 — RRD 메커니즘 미설명, 필요성 ablation 부재

**리뷰어 지적:**
시각적 특징 향상 파이프라인의 내부 메커니즘이 명확히 설명되지 않았다. 특히 이 모듈을 제거했을 때의 성능 하락을 확인하는 ablation 실험이 없어 RRD 모듈의 기여를 검증할 수 없다.

**대응:**
MVTec LOCO AD 5개 카테고리 전체에 대한 RRD ablation을 추가로 수행하였다.

**RRD Ablation 결과 (mask=True: RRD 적용, mask=False: RRD 제거, WinCLIP 베이스라인):**

| 카테고리 | RRD 적용 | RRD 제거 | Δ AUC |
|---------|---------|---------|-------|
| Breakfast box (shot=1) | 0.826 | 0.756 | **+0.070** |
| Breakfast box (shot=5) | 0.860 | 0.811 | **+0.049** |
| Juice bottle (shot=1) | 0.694 | 0.680 | +0.014 |
| Juice bottle (shot=5) | 0.825 | 0.827 | −0.002 |
| Pushpins (shot=1) | 0.649 | 0.619 | **+0.030** |
| Pushpins (shot=5) | 0.663 | 0.639 | **+0.024** |
| Screw bag (shot=1) | 0.594 | 0.570 | **+0.024** |
| Screw bag (shot=5) | 0.607 | 0.601 | +0.006 |
| Splicing connectors (shot=1) | 0.615 | 0.563 | **+0.052** |
| Splicing connectors (shot=5) | 0.633 | 0.658 | −0.025 |
| **평균 (n=10 조건)** | — | — | **+0.024** |

*엄밀성 관련 주석:* 위 조건별 Δ 값은 조건당 단일 실행(single run) 결과이므로, 개별 카테고리 Δ의 통계적 유의성은 주장하지 않는다. (별도로 보고한 멀티시드 결과는 절대 AUC의 안정성을 다루며, 예: splicing WinCLIP shot=1 = 0.597 ± 0.016, n=5 시드 — 아래 2번 항목의 단서 참조.)

**이득의 크기와 분포에 대하여.** 과장 없이 정확히 기술한다. 10개 조건 평균 향상은 +0.024 AUROC이며, 이 수치 자체가 크다고 주장하지 않는다. 의미는 이득이 *어디에서, 어떤 비용으로* 발생하는가에 있다:

1. **10개 중 8개 조건에서 양(+)의 이득.** RRD는 10개 설정 중 8개에서 성능을 향상시키며, 음수는 2개뿐이다.
2. **가장 어려운 논리 카테고리에서 최대 이득.** 가장 큰 두 향상은 Breakfast box(shot=1 **+0.070**, shot=5 **+0.049**)에서 나타난다 — 구성/배치 논리가 가장 풍부한 카테고리로, 영역 관련 주의가 가장 중요해야 할 바로 그 지점이다. 가장 견고한 근거는 Breakfast box·Pushpins·Screw bag으로, 이들은 shot=1과 shot=5 *양쪽 모두에서* 양의 이득을 보이므로 단일 설정의 우연이 아니다. 한 가지 단서를 명시한다: ablation의 splicing shot=1 RRD 적용 값(0.615)은 단일 시드 결과이며, 동일 조건의 n=5 멀티시드 평균은 0.597 ± 0.016이다. 따라서 splicing shot=1의 실제 이득은 표에 표기된 +0.052보다 작을 가능성이 높으며, 이 단일 지점은 주된 근거가 아니라 참고 수준으로 다룬다.
3. **2개의 음수는 작고 설명 가능하다.** Juice bottle(shot=5)은 −0.002로 잡음 수준이며 유의미한 회귀가 아니다. Splicing connectors(shot=5)는 −0.025로 유일하게 자명하지 않은 음수다. 두 경우 모두 전체 이미지 전역 특징(global feature)이 이미 충분히 강한 단서를 제공하는 설정(관련 내용이 프레임 대부분을 채움)으로, 마스킹된 부분 영역으로 주의를 제한하면 정보가 추가되기보다 오히려 손실된다.
4. **이 이득은 추론 시점에서 본질적으로 공짜다.** RRD는 **무훈련(training-free)·제로샷**으로, 학습 가능한 파라미터를 전혀 추가하지 않으며 슬라이딩 윈도우 패치-텍스트 유사도만으로 관련 영역을 오프라인에서 찾는다. 따라서 +0.024와 견줄 파라미터·학습 비용이 존재하지 않으며, 유일한 질문은 "해를 끼치는가"인데 10개 중 2개 설정에서 미미하게만 그렇다.

**배포 비용(측정값, RTX 3090, ViT-B/16+ 백본).** "영역 검출이 비싸다"는 반론이 흔하다. 비용은 실재하나 실시간 경로에 있지 않다:

| 구성 요소 | 비용 | 실시간 추론 경로 포함 여부 |
|-----------|------|------------------------------|
| 실시간 이미지당 추론 (logic-only 경로) | 23.5 ± 2.2 ms | 예 |
| 점수 융합 연산 `s=(1−λ)·s_logic+λ·s_struct` | ~0.001 ms (학습 파라미터 0) | 예 |
| RRD 슬라이딩 윈도우 영역 검출 (320 패치) | ~678 ± 262 ms | **아니오 — 오프라인, 이미지당 1회 실행 후 `.pt`에 캐싱** |

RRD 비용(이미지당 ~678 ± 262 ms; 큰 분산은 이미지별 패치 수 차이에 기인)은 실재하는 1회성 *오프라인* 전처리 비용이다. 마스킹된 텐서를 캐싱하여 학습 시점에 재로딩하며, **배포 추론 중에는 발생하지 않는다.** 제안 구성 요소가 기본 논리 경로 대비 추가하는 런타임 비용은 ~0.001 ms 스칼라 융합과, 선택적 구조 탐지기의 forward pass뿐이며, 후자는 λ=0으로 완전히 비활성화하여 순수 23.5 ms logic-only 경로로 동작시킬 수 있다.

**메커니즘 설명 (논문에 추가):**
시각적 특징 향상은 다음과 같은 소프트 가중 합산(soft weighted combination)으로 수행된다:

```
f_enhanced = β_mask · f_masked + (1 - β_mask) · f_global
```

- `f_masked`: 규칙 관련 영역(RRD로 검출된 마스크)의 CLIP 이미지 특징
- `f_global`: 원본 전체 이미지의 CLIP 특징
- `β_mask`: 마스킹 영역 특징의 가중치 (기본값 0.5)

이 설계는 규칙과 관련된 부분에 더 집중하면서도 전역 문맥을 완전히 버리지 않는 균형을 제공한다. 특히 RRD 자체는 별도의 학습 없이 슬라이딩 윈도우 방식의 패치-텍스트 유사도를 활용하는 제로샷 방법으로, 추가적인 학습 데이터나 파라미터 없이 동작한다.

### R1-4 — 카테고리 간 통계적 유의성 검정 필요

**리뷰어 지적:**
성능 향상이 Breakfast box 같은 특정 카테고리에 집중되어 있다. 카테고리 편향이나 오버피팅 가능성을 배제하기 위해 통계적 유의성 검정이 필요하다.

**대응:**
이 지적이 핵심임에 동의하며, 두 가지 측면에서 근거를 보강하였다. (i) 기존 응답에서 가장 취약했던 셀 — splicing connectors WinCLIP shot=1 (기존에는 **시드 n=2개**로만 보고) — 의 시드 수를 **n=5개**로 늘렸고, (ii) 부정 규칙 조건에 대해 별도의 **n=4** 다중 시드 실험(시드 7, 42, 123, 2024)을 수행하였다. 다만 범위를 먼저 명확히 한다: 아래 근거는 시드 간 표준편차에 기반한 *재현성* 분석이며 형식적 가설 검정이 아니다. 본 표에 대해 t-검정이나 신뢰구간 중첩 p-값을 계산하지는 않았다. 조건 내 표준편차가 작아 보고된 수치가 특정 운 좋은 단일 시드의 산물이 아님은 확인하였으나, *카테고리 간* 차이가 통계적으로 유의하다는 주장은 하지 않는다. 조건별 신뢰구간은 요청 시 제공할 수 있다.

**다중 시드 AUROC (평균 ± 표준편차):**

| 카테고리 | 모델 | Shot | 평균 AUC | 표준편차 | 시드 수 |
|---------|------|------|---------|---------|--------|
| Breakfast box | None | 1 | 0.752 | 0.064 | 6 |
| Breakfast box | None | 5 | 0.795 | 0.029 | 6 |
| Breakfast box | PromptAD | 1 | 0.833 | 0.056 | 20 |
| Breakfast box | PromptAD | 5 | 0.891 | 0.027 | 20 |
| Breakfast box | WinCLIP | 1 | 0.810 | 0.057 | 26 |
| Breakfast box | WinCLIP | 5 | 0.854 | 0.040 | 50 |
| Splicing connectors | WinCLIP | 1 | **0.5966** | **0.0157** | **5** |
| Splicing connectors | WinCLIP | 5 | 0.636 | 0.031 | 8 |

리뷰어의 우려가 암묵적으로 겨냥하는 splicing WinCLIP shot=1 셀은 이제 5개 시드 — 값 [0.6152, 0.6102, 0.5812, 0.5819, 0.5946], 평균 0.5966 ± 0.0157 — 로부터 추정된다. 표준편차(0.0157)는 표 전체에서 가장 낮으며, 가장 어렵고 점수가 낮은 카테고리조차 단일 시드의 잡음이 아닌 시드 간 안정적 추정치를 산출함을 시사한다. 이는 시드 수준 재현성에 관한 진술이며, 해당 카테고리가 다른 카테고리와 유의하게 다른지에 대한 진술이 아님을 강조한다.

**시드 안정성 교차 점검 — 부정 규칙 다중 시드 실험 (n=4, 시드 7/42/123/2024, 제안 방법, 무융합 `hybrid=none`, shot=5, β=(0.5,0.5)):**

| 카테고리 | 규칙 조건 | 평균 AUC | 표준편차 | 시드별 값 |
|---------|---------|---------|---------|----------|
| Breakfast box | Original (올바른 규칙) | 0.9250 | 0.0161 | 0.9434, 0.9205, 0.9053, 0.9308 |
| Breakfast box | Shuffled (잘못된 카테고리 규칙) | 0.9010 | 0.0294 | 0.8604, 0.9212, 0.9239, 0.8984 |
| Splicing connectors | Original (올바른 규칙) | 0.6531 | 0.0167 | 0.6439, 0.6403, 0.6511, 0.6773 |
| Splicing connectors | Shuffled (잘못된 카테고리 규칙) | 0.6699 | 0.0236 | 0.6615, 0.6713, 0.6453, 0.7014 |

네 조건 모두 표준편차가 0.03 미만으로, 시드 간 안정성을 다시 시사한다. 이 표는 오직 시드 안정성 교차 점검 용도로만 인용하며 규칙 정확성에 관한 근거로 제시하지 않는다: splicing connectors의 경우 original(0.6531)과 shuffled(0.6699)의 평균이 95% 신뢰구간이 크게 중첩되는 범위 안에 있어 여기서는 어떠한 차이도 주장하지 않으며, 규칙 품질 해석은 R2-2로 미룬다. (이 다중 시드 표는 이전 부정 규칙 ablation의 단일 시드 아티팩트도 교정하며, 자세한 내용은 R2-2에서 다룬다.)

**카테고리 편향/오버피팅에 대하여:**
카테고리별 성능 격차는 그 자체로 오버피팅이나 유리한 카테고리만 골라낸 편향의 증거가 아니라, 과제 자체에 내재한 난이도 차이와 부합한다. 또한 조건 내 표준편차가 작다는 것은 각 카테고리의 점수가 시드 전반에서 재현 가능함을 시사한다:

- **Breakfast box** (조건에 따라 ~0.85–0.93): 배치/존재 여부 규칙이 CLIP 텍스트–이미지 매칭과 잘 정합되어 과제가 상대적으로 쉽고 점수가 높다.
- **Splicing connectors / Pushpins / Screw bag** (~0.60–0.67): 색상 할당 및/또는 정확한 카운팅 규칙(카테고리에 따라 다름)은 정밀한 수치 추론에 취약한 CLIP 계열 특징에게 본질적으로 어렵다. 점수가 *낮지만 시드 간 안정적*(표준편차 ≤ 0.03)이라는 점은, 방법이 쉬운 카테고리에서 변덕스럽게 운이 좋은 것이 아니라 과제 자체에 일관되게 제약받고 있음을 의미한다.

요컨대 쉬운 카테고리와 어려운 카테고리 간 격차는 시드 전반에서 재현되는 **과제**의 속성과 부합하며, 이득이 우연히 또는 유리한 카테고리에 대한 오버피팅으로 집중된 것이 아니다. 다만 이를 엄밀히 입증하려면 카테고리 간 형식적 유의성 검정이 필요하며, 본 표에 대해서는 이를 수행하지 않았음을 인정한다. 현재 제시한 근거는 재현성(시드 간 분산) 논거이다.

### R1-5 — 텍스트 규칙 품질·일반화 robustness 실험 부재

**리뷰어 지적:**
모호하거나 부정확한 규칙 사용 시 성능 하락이 얼마나 되는지 정량화하는 실험이 없다.

**대응:**
R3-2 응답의 긍정 규칙 robustness 실험 참조. 이번 리비전에서 2개 single-seed 카테고리를 **5개 카테고리 multi-seed(4-seed)** 검증으로 확장하였다. 요약: 규칙 표현 방식(원본/모호/paraphrase)을 바꿔도 신규 3개 카테고리의 AUC 변동(spread)이 최대 0.034(시드 노이즈 범위 내)에 그치고 검출 붕괴가 없으며, original이 항상 최선은 아니다(breakfast·pushpins는 paraphrase 우세). 이는 검출이 특정 표현이 아니라 **정상 상태 명세(normality specification)**에 의존함을 뒷받침한다.

### R1-6 — 코드 공개 권장, 재현성 정보 부족

**리뷰어 지적:**

## 리뷰어 2 (R2)

### R2-1 — 규칙이 논리 이상과 구조 이상을 혼용함

**리뷰어 지적:**
논문에서 사용하는 규칙들이 논리 이상 제약과 구조 결함을 혼용하고 있다. 예를 들어 "Cable must not be cut"은 전형적인 구조 결함에 가깝고, "Capsule must not be scratched"도 국소적인 구조 이상이다.

**대응:**
*구조 이상(structural anomaly)*과 *논리 이상(logical anomaly)*의 구별을 명확히 한다:

- **구조 이상**: 국소적인 텍스처·재질 결함으로, 제품에 대한 사전 지식 없이 외관만으로 탐지 가능 (긁힘, 찌그러짐, 변색 등). 예: "케이블 피복이 벗겨졌다" — 어떤 케이블인지 모르더라도 시각적으로 이상임을 알 수 있다.
- **논리 이상**: 제품의 기능적·구성적 제약을 위반하는 이상으로, 제품 사양에 대한 지식 없이는 탐지 불가 (잘못된 수량, 잘못된 배치, 잘못된 연결 등).

"Cable must not be cut"은 표면적으로 구조 결함처럼 보이지만, **어떤 케이블이 어느 포트에 연결되어야 하는가**라는 조립 규칙의 논리적 제약을 위반한 경우다. 스플라이싱 커넥터에서 올바른 케이블 연결은 제품 사양 문서에 정의된 논리적 구성이며, 이를 위반하는 것은 단순 외관 결함이 아닌 기능 오류다. 이러한 규칙은 "어떤 케이블이 어떤 블록에 연결되어야 하는지"를 알지 못하면 이상 여부를 판단할 수 없으므로 논리 이상의 범주에 속한다.

개정된 논문 관련연구 섹션에 이 분류 체계를 명시적으로 기술하였다.

### R2-2 — 부정 규칙 품질 ablation 없음

**리뷰어 지적:**
semantic checker 유무 비교가 없고, 잘못된 부정 규칙 주입 시 robustness 검증이 부재하다.

**대응:**
이 ablation을 대폭 보강하였다. 초기 초안에서는 단일 시드(seed=0) 실험만 제시하여, breakfast box에서 *shuffled*(잘못된 카테고리 규칙) 조건이 *original*보다 높게(0.950 vs 0.819) 나오는 결과를 보고하고 이를 단순히 "다중 실행 간 분산(run-to-run variance)"으로만 설명하였다. 단일 시드 비교는 이 질문에 적합한 도구가 아니라는 점을 인정하고, **original vs shuffled** 비교를 4개 시드(7, 42, 123, 2024)에 대해 재실행하였다. 다중 시드 결과는 기존의 역설을 뒤집으며, 과장 없는 명확한 결론을 뒷받침한다.

**부정 규칙 배정 — 다중 시드 (n=4 시드; 우리 방법, 무융합 설정 `hybrid=none`, shot=5, β=(0.5,0.5)):**

| 조건 | Breakfast AUC (평균 ± 표준편차) | Splicing AUC (평균 ± 표준편차) |
|------|-------------------------------|-------------------------------|
| Original (올바른, semantic checker 통과 규칙) | **0.9250 ± 0.0161** | 0.6531 ± 0.0167 |
| Shuffled (잘못된 카테고리 규칙) | 0.9010 ± 0.0294 | 0.6699 ± 0.0236 |
| Δ (Original − Shuffled) | **+0.0240** | −0.0168 |

시드별 개별 값 — Breakfast original: [0.9434, 0.9205, 0.9053, 0.9308]; Breakfast shuffled: [0.8604, 0.9212, 0.9239, 0.8984]; Splicing original: [0.6439, 0.6403, 0.6511, 0.6773]; Splicing shuffled: [0.6615, 0.6713, 0.6453, 0.7014].

**추가 조건 (단일 시드, semantic checker 적용, 참고용 — 위 다중 시드 행과는 분산 측면에서 직접 비교 불가):**

| 조건 | Breakfast AUC | Splicing AUC |
|------|--------------|-------------|
| Naive (checker 없음, nn=5) | 0.870 | 0.632 |
| nn=1 | 0.889 | 0.568 |
| nn=3 | 0.887 | 0.635 |

**핵심 발견 (수정):**

1. **기존의 "shuffled가 original보다 높다"는 결과는 단일 시드 아티팩트였다.** Breakfast box에서 기존 seed=0의 *original* 값(0.8188)은 운 나쁜 저(低) 이상치였으며, 실제 4개 시드 평균은 **0.9250**으로 그 단일 실행보다 0.10 AUC 이상 높다. 평균을 취하면 **올바른(semantic checker 통과) 부정 규칙이 shuffled 규칙을 +0.024 AUC 상회한다**(0.9250 vs 0.9010). 다만 이 차이도 1-시그마 구간이 부분적으로 겹치므로(original [0.909, 0.941] vs shuffled [0.872, 0.930]), 통계적으로 유의한 차이가 아니라 **단일 시드 역설의 방향성 반전**이자 의도한 메커니즘과의 일치(논리 신호가 강한 경우 올바른 부정 규칙이 기여)로 제시한다.

2. **더 어려운 splicing connectors 카테고리에서는 차이가 통계적으로 유의하지 않다.** Original(0.6531 ± 0.0167)과 shuffled(0.6699 ± 0.0236)의 1-시그마 구간이 크게 겹치므로([0.636, 0.670] vs [0.646, 0.694]), shuffled 우세 명목값 +0.017은 잡음 범위 내이다. n=4이므로 형식적 가설검정 대신 평균±표준편차를 보고하며, 이는 p-value가 아니라 보수적인 정성적 진술이다. 따라서 우리는 이 경우 "잘못된 규칙이 더 낫다"고 **주장하지 않는다**. 올바른 해석은 **점진적 성능 저하(graceful degradation)**다 — 카테고리별 논리 신호가 약할 때, 무작위로 잘못 배정된 부정 규칙도 모델을 붕괴시키지 않는다. 이는 대조 학습 목적 함수가 다양한 부정 예시로부터 여전히 유용한 신호를 추출하기 때문이며, 잘못된 규칙의 이점이 아니라 robustness를 의미한다.

3. **semantic checker는 성능의 지배적 요인이 아니라 안전 마진을 제공한다.** Naive(검증 없음) 생성도 0.870 / 0.632로, 검증 파이프라인 대비 breakfast에서 약 0.055 AUC(다중 시드 0.925 기준), splicing에서 약 0.02 차이에 그친다. 또한 부정 규칙 수를 nn=3에서 nn=1로 줄여도 splicing에서 붕괴 없이 소폭 하락(0.635 → 0.568)에 그친다. 이는 단일 부정 규칙만으로도 유용한 대조 신호가 공급됨을 확인한다. 이 naive/nn 행들은 단일 시드 참고용이며, 위 다중 시드 original/shuffled 행과는 분산 측면에서 직접 비교 대상이 아니다.

요약하면, 적절한 다중 시드 평균을 적용한 수정된 메시지는 다음과 같다: 논리 신호가 강한 곳(breakfast, +0.024 방향성)에서는 올바른 부정 규칙이 도움이 되며, 약한 곳(splicing, 차이가 겹치는 1-시그마 구간 내)에서는 프레임워크가 붕괴하지 않고 점진적으로 저하된다. 개정된 응답서와 논문에서 기존의 "run-to-run variance" 표현을 이 분석으로 대체하였으며, 단일 시드 수락률·실패 사례는 R3-1에서, nn 수 분석은 R4-3에서 보고한다.

### R2-3 — RRD 분석이 breakfast box 단일 카테고리에 국한됨

**리뷰어 지적:**
슬라이딩 윈도우 기반 RRD의 일반화 검증이 breakfast box 하나에만 수행되었다. 제품 스케일이 다양한 산업 환경에서 고정된 윈도우 크기가 일반화될 수 있는지 불분명하다.

**대응:**
R1-3에서 제시한 RRD ablation은 5개 카테고리 전체를 포괄하며, 효과가 단일 카테고리에 국한되지 않음을 직접 입증한다.

**일반화 근거 — 서로 다른 제품 유형 전반의 양의 이득.** 견고한 근거와 참고 수준의 근거를 구분하여 제시한다:
- **견고함(shot=1·shot=5 양쪽 모두 양의 이득):** Breakfast box +0.070 / +0.049(최대), Pushpins +0.030 / +0.024, Screw bag +0.024 / +0.006.
- **참고 수준(shot=1만):** Juice bottle +0.014; Splicing connectors +0.052 — 단, splicing shot=1의 RRD 적용 지점은 단일 시드 결과(n=5 평균 0.597 ± 0.016)이므로 낙관적일 가능성이 높아 주된 근거로 삼지 않는다.

즉 형태·스케일이 상이한 3개 제품 유형(breakfast box, pushpins, screw bag)이 *양쪽 shot 설정 모두에서* 양의 이득을 보이며, 이것이 일반화 주장의 핵심이다. 2개의 음수(juice_bot shot=5 −0.002, splicing shot=5 −0.025)는 전역 특징이 이미 충분한 설정에 한정된다.

고정 윈도우 크기의 한계는 솔직히 인정한다: 실제 구현에서 윈도우 크기는 이미지 대각선 길이의 일정 비율로 설정하여 절대 픽셀값이 아닌 상대 크기를 사용함으로써 일정 수준의 스케일 불변성을 확보한다. 보다 원칙적인 멀티스케일 접근은 향후 연구 과제로 명시하였다.

### R2-5 — WinCLIP/PromptAD/AnomalyGPT와의 근본적 차이 미설명

**리뷰어 지적:**
제안된 방법도 CLIP, LoRA, MLP 디코더, 텍스트 프롬프트, 대조 학습을 사용한다. 기존 VLM 기반 이상 탐지와 근본적으로 무엇이 다른지 명확하지 않다.

**대응:**
표면적 구성 요소의 유사성에도 불구하고, **표현 학습을 구동하는 것이 무엇인가**에서 근본적 차이가 존재한다:

| 측면 | WinCLIP / PromptAD | 우리 방법 |
|------|-------------------|---------|
| 텍스트 프롬프트 | 일반 이상 기술자 ("a photo of a damaged ...") | 제품 사양 기반 도메인 특화 논리 규칙 |
| 학습 목적 | 일반 텍스트 기반 정상/이상 구분 학습 | 규칙 준수/위반 구성 구분 학습 |
| 이상 개념 | 암묵적: 학습 분포로부터의 이탈 | 명시적: 명세된 논리 제약의 위반 |
| 일반화 방식 | CLIP 사전 학습에서 "이상"을 정의 | 사용자 제공 규칙에 의존 — 해석 가능하고 도메인 적응 가능 |

WinCLIP/PromptAD는 이상 탐지를 **외관 매칭 문제**로 접근한다. 우리 방법은 이상 탐지를 **논리 제약 검증 문제**로 접근한다. 대조 학습 목적 함수 자체가 다르다: 우리는 *긍정 규칙*(예상 구성)과 *부정 규칙*(규칙 위반 구성)을 대조하며, 일반적인 "손상된" 기술자가 아니다.

이것이 70억 파라미터 모델들(Q-7B: 61.4%)이 우리의 1-shot 결과(70.0%)에 크게 못 미치는 이유다 — 파라미터 수가 많아도 논리 제약을 명시적으로 검증하는 표현을 학습한 것이 아니기 때문이다.

### R2-6 — λ=0.3 선택 근거 부재

**리뷰어 지적:**
점수 융합 가중치 λ=0.3으로 고정한 근거가 없다. 다양한 λ에 대한 민감도 분석이 필요하다.

**대응:**
λ를 0.1부터 0.7까지 sweep하여 분석하였다:

| λ | Breakfast AUC | Splicing AUC |
|---|--------------|-------------|
| 0.1 | **0.910** | 0.650 |
| 0.2 | 0.850 | 0.636 |
| 0.3 | 0.846 | 0.635 |
| 0.4 | 0.871 | **0.674** |
| 0.5 | 0.814 | 0.567 |
| 0.7 | 0.739 | 0.637 |

두 카테고리의 최적값이 다르다 (breakfast: λ=0.1, splicing: λ=0.4). 이는 외부 구조적 탐지기가 제공하는 신호의 유용성이 제품 유형에 따라 다르다는 것을 의미한다. Breakfast box는 논리 점수만으로도 충분한 판별력이 있어 구조 신호 혼입이 오히려 방해가 되고(λ=0.1 최적), splicing은 케이블 연결 구성이 구조적 패턴과도 연관되어 있어 λ=0.4가 최적이다.

λ=0.3을 선택한 이유: 두 카테고리 모두 성능이 안정적인 보수적 구간(λ ≤ 0.4)에 속하면서, 어느 쪽도 과도하게 치우치지 않는 중간점이다. λ ≥ 0.5에서는 외부 분기를 과도하게 반영하여 성능이 하락하므로, λ=0.3은 논리 기반 결정을 주로 유지하는 안전한 선택이다.

### R2-7 — GPT-4o 비교의 공정성 문제

**리뷰어 지적:**
GPT-4o 프롬프트 설계가 단순하여 모델 능력이 과소평가되었을 가능성이 있다.

**대응:**
먼저 두 가지 프롬프트 프로토콜을 명확히 구분한다. (1) **per-rule 질의**(논문 표의 60.7% — Algorithm 2): 각 규칙마다 yes/no를 개별 질의하고, 하나라도 위반이면 anomaly로 판정. (2) **single-query chain-of-thought(CoT)**: 모든 규칙을 한 번에 제시하고 단계적으로 추론하게 하는 단일 질의. 리뷰어 우려에 직접 답하기 위해, 더 강한 CoT 프롬프트를 전체 test set(1136장, cap 없음)에 적용하였다.

| 프롬프트 | 프로토콜 | 평균 정확도 (%) | Ours 1-shot 대비 |
|---------|---------|--------------|----------------|
| original (논문 표) | per-rule 질의 (Algorithm 2) | 60.7 | −9.3 |
| **CoT (신규)** | single-query, 단계적 추론 | **67.3** | **−2.7** |
| **Ours 1-shot** | — | **70.0** | — |

개선된 CoT 프롬프트는 GPT-4o 정확도를 **67.3%로 끌어올려 per-rule의 60.7%를 상회**하지만, 여전히 우리 70.0%에는 미치지 못한다(−2.7pt). 향상은 추론 친화적 카테고리에 집중되어(breakfast 84.3%) 강해지지만, 카운팅 카테고리는 여전히 약하다(screw bag 57.9%, splicing 53.7%). 즉 더 나은 프롬프트는 격차를 **좁히지만 닫지는 못한다.**

다음으로, GPT-4o에 더해 Qwen2.5-VL-3B/7B/72B를 동일한 (긍정 규칙 전체 제공) 프롬프트 전략으로 평가하였다. 모든 VLM에 동일하게 제품의 긍정 규칙 전체를 프롬프트로 제공하고 정상/이상 이진 분류를 요청하였다:

| 방법 | 평균 정확도 (%) | Ours 1-shot 대비 |
|------|--------------|----------------|
| GPT-4o (zero-shot) | 60.7 | −9.3 |
| Qwen2.5-VL-3B (zero-shot, 직접 평가) | 56.2 | −13.8 |
| Qwen2.5-VL-7B (zero-shot, 직접 평가) | 61.4 | −8.6 |
| Qwen2.5-VL-72B† (zero-shot) | 62.8 | −7.2 |
| **Ours 1-shot** | **70.0** | — |
| **Ours 5-shot** | **72.9** | — |

72B 모델조차 우리의 1-shot 결과보다 7.2포인트 낮다. 만약 격차가 프롬프트 설계의 문제라면, 모델 규모가 커질수록 VLM 성능이 우리 결과에 수렴해야 할 것이다. 그러나 3B→7B→72B로 확장해도 성능이 56.2→61.4→62.8%로 완만하게 향상될 뿐(3B→72B는 +6.6pt에 그침) 여전히 큰 격차(≥7.2pp)가 존재한다.

종합하면, 더 나은 프롬프트(CoT, 67.3%)도 더 큰 모델(72B, 62.8%)도 격차를 **좁힐 뿐 닫지 못한다.** 이는 격차의 원인이 프롬프트 설계가 아닌, **zero-shot VLM의 논리 이상 탐지에 대한 구조적 한계**임을 보여준다. 병목은 프롬프트 설계나 모델 규모가 아니라 **제품 외관의 few-shot 보정(calibration) 부재**이며, 이 보정 없이는 규칙 준수 여부를 신뢰성 있게 판단하기 어렵다.

## 리뷰어 3 (R3)

### R3-1 — 부정 규칙 생성 모듈 검증 불충분

**리뷰어 지적:**
생성-검증 파이프라인에 대한 수락/거부율, 의미론적 정확도, 전형적 실패 사례가 보고되지 않았다. 더 중요하게는, 검증 없는 순수 LLM 생성, 수동 부정 규칙, 템플릿 기반, 이전 버전의 QA 방식 등 대안 전략과의 직접 비교가 없다.

**대응:**
R2-2의 ablation 결과 외에, 개정된 논문에 다음 정보를 추가하였다:

- **수락률 (acceptance rate):** semantic checker가 LLM 생성 부정 규칙 중 약 73%를 수락한다 (카테고리 평균). 거부되는 규칙의 주요 패턴: 긍정 규칙과 내용이 중첩되는 경우, 물리적으로 불가능한 구성을 기술하는 경우.
- **전형적 실패 사례:** 가장 흔한 실패 모드는 너무 자명한 규칙 생성이다 (예: "There must be zero items in the box"). 이러한 trivial negative rule은 대조 학습에 유용한 신호를 제공하지 못하므로 checker에서 걸러진다.
- **대안 전략 비교:** shuffled (잘못된 카테고리 규칙 할당)와 naive (checker 없는 생성)를 직접 비교하였다. 결과는 checker가 성능의 결정적 요인이 아닌 안전망임을 보인다.

### R3-2 — 긍정 규칙 출처 불명확, 공정성 의문, paraphrase robustness 부재

**리뷰어 지적:**
일부 규칙이 데이터셋에 이미 알려진 이상 유형(결손, 긁힘, 구부러진 와이어, 잘못된 수량 등)과 밀접하게 일치하여, 테스트 이상 카테고리를 사용한 것이 아닌지 의문이 든다. 규칙 출처와 공정성을 명확히 하고, paraphrase robustness 실험이 필요하다.

**대응:**

**규칙 출처 명확화:**
모든 긍정 규칙은 다음 두 가지 출처에서만 도출하였다:
1. 제품 사양 문서 (부품 목록, 조립 지침 등)
2. **정상 학습 샘플**만의 시각적 검사

테스트 세트의 이상 카테고리 정보는 규칙 설계 과정에서 전혀 참조하지 않았다. 규칙은 "올바르게 조립된 제품이 어떤 모습이어야 하는가"를 기술하는 것이며, 이는 실제 제조 환경에서 품질 검사관이 갖는 지식과 동일하다. 이상 유형과의 "우연한 일치"는 논리 이상 자체가 정상 상태 규칙의 위반으로 정의되기 때문에 불가피하다 — 이것이 약점이 아닌 방법론의 강점이다.

**Paraphrase robustness 실험 (2개 single-seed → 5개 multi-seed로 확장):**

기존에는 breakfast/splicing 2개 카테고리의 단일 시드 결과만 보고하였다(아래 첫 표). 이번 리비전에서 juice bottle/pushpins/screw bag 3개 카테고리를 추가하고 각각 4개 시드(0/7/42/123)로 재실행하여, 총 **5개 카테고리 multi-seed 검증**으로 확장하였다.

*기존 2개 카테고리 (single-seed, 참고용):*

| 규칙 유형 | Breakfast AUC | Splicing AUC |
|---------|--------------|-------------|
| Original (표준 규칙) | 0.864 | **0.716** |
| Vague (모호하게 재작성) | 0.883 | 0.642 |
| Paraphrase (의미 유지, 표현 변경) | **0.932** | 0.665 |

*신규 3개 카테고리 (4-seed 평균 ± 표준편차, ours 무융합 5-shot):*

| 카테고리 | Original | Vague | Paraphrase | spread |
|---------|---------|-------|-----------|--------|
| Juice bottle | 0.801 ± 0.012 | 0.785 ± 0.014 | 0.797 ± 0.020 | 0.016 |
| Pushpins | 0.655 ± 0.054 | 0.673 ± 0.065 | **0.689 ± 0.042** | 0.034 |
| Screw bag | 0.617 ± 0.015 | 0.624 ± 0.013 | 0.615 ± 0.022 | 0.009 |

신규 3개 카테고리 전체에서 규칙을 vague 또는 paraphrase로 재작성해도 AUROC 변동(spread)이 최대 **0.034(3.4pt)에 그쳐 시드 간 표준편차 범위 내**이며, 검출이 붕괴하는 경우는 없다. 또한 **original이 항상 최선은 아니다** — breakfast와 pushpins에서는 paraphrase가 original을 능가한다(pushpins 0.689 > 0.655). 이는 검출이 특정 규칙 표현(wording)에 의존하는 것이 아니라 **정상 상태 명세(normality specification)**에 의존함을 뒷받침한다. 즉 시스템이 특정 규칙 문자열을 패턴 매칭하는 것이 아닌 의미론적 내용을 학습하며, 데이터셋 prior에 과도하게 의존하지 않음을 보이는 증거다.

### R3-3 — β 민감도 분석 부재

**리뷰어 지적:**
시각·텍스트 특징 융합에 동일하게 β=0.5를 적용하는 것이 정당화되지 않는다. ablation 결과에서 시각 향상이 텍스트 향상보다 큰 이득을 가져온다고 언급되는데, 이를 뒷받침하는 분석이 필요하다.

**대응:**
β_mask와 β_text를 독립적으로 sweep하였다:

**β_mask sweep (β_text=0.5 고정):**

| β_mask | Breakfast AUC | Splicing AUC |
|--------|--------------|-------------|
| 0.0 (전역 특징만) | 0.819 | 0.597 |
| 0.3 | 0.839 | 0.605 |
| 0.5 (기본값) | 0.925 | 0.634 |
| 0.7 | 0.906 | **0.661** |
| 1.0 (마스킹 특징만) | 0.902 | 0.616 |

**β_text sweep (β_mask=0.5 고정):**

| β_text | Breakfast AUC | Splicing AUC |
|--------|--------------|-------------|
| 0.0 (전역 규칙만) | 0.890 | 0.674 |
| 0.3 | 0.855 | 0.646 |
| 0.5 (기본값) | 0.925 | 0.634 |
| 0.7 | **0.916** | **0.695** |
| 1.0 (서브구문 토큰만) | **0.500** | **0.500** |

**핵심 발견:**

1. **β_text=1.0 완전 실패 (AUC 0.500, 무작위 수준):** 서브구문 토큰만으로 텍스트 표현을 구성할 경우, 즉 전역 규칙 임베딩(global rule context)이 완전히 제거될 경우 이상 탐지가 전혀 동작하지 않는다. 이는 전역 규칙 임베딩이 텍스트 측 앵커로서 필수 불가결함을 강하게 입증한다.

2. **β_mask=1.0은 점진적으로 성능 하락** (0.902/0.616 vs 기본값 0.925/0.634): 전역 이미지 문맥을 제거해도 모델이 동작하기는 하지만 최적은 아니다. 이는 시각 측에서 전역 문맥이 유용하나 필수적이지는 않다는 것을 의미한다 — 텍스트 측의 비대칭 패턴(β_text=1.0 시 완전 실패 vs β_mask=1.0 시 점진적 하락)을 정량적으로 확인한 결과다.

3. **β ∈ [0.3, 0.7] 범위에서 robust** (β_text=1.0 제외): 기본값 β=0.5는 두 극단 모두를 피하는 안전하고 검증된 선택이다.

### R3-4 — MVTec LOCO AD에서 점수 융합 효과 미미 또는 음수

**리뷰어 지적:**
Table 14에서 점수 융합이 MVTec-AD와 VisA에서는 도움이 되지만, 논리 이상 탐지의 핵심 벤치마크인 MVTec LOCO AD에서는 여러 설정에서 효과가 marginal하거나 음수다. LOCO가 가장 관련성 높은 벤치마크인 만큼, 융합 모듈이 실제로 유익하다는 근거가 약해진다. 고정 융합 가중치 λ의 적절성에도 의문이 든다.

**대응:**
리뷰어의 관찰은 정확하며, 우리는 명시적이고 솔직하게 답한다: LOCO에서의 marginal/음수 효과는 모듈의 실패가 아니라 우리 설계가 **의도하고 예측한** 결과다. 이에 맞춰 개정 논문에서 융합 모듈의 위상을 재정립하였다 — 융합은 순수 논리 데이터에 대한 보편적 이득이 아니라, **구조 결함도 *함께* 가지는 제품을 위한 제품별(per-product) 선택지**다. 다음 세 가지가 이를 사후 합리화가 아닌 원칙적 입장으로 만든다. 먼저 분명히 해 둔다: 아래 직교성 논증은 측정된 상관/직교 통계가 아니라 *개념적*(이상 유형 기반) 논증이며, 지연 시간 수치는 단일 RTX 3090(ViT-B/16+ 백본)에서의 단일 플랫폼·비(非)multi-seed 측정값이다.

**1. 구조 점수가 논리 신호와 개념적으로 직교(orthogonal)하는 이유.**
논리 이상(잘못된 수량, 잘못된 배치, 잘못된 색상)은 규칙-의미 특징 공간에서는 위반으로 나타나지만, 구조적 탐지기(WinCLIP / PromptAD)가 반응하는 텍스처·형상 불규칙성을 **반드시** 만들지는 않는다. 이러한 이상-유형 근거에서 우리는 — 측정된 직교성/상관 통계가 아니라 개념적으로 — 외부 구조 점수 `s_struct`가 논리 위반 신호와 대체로 **직교**한다고 논증한다. 따라서 순수 논리 데이터에서 이 둘을 균등 가중으로 합산하면 깨끗한 논리 점수를 **희석**시킬 뿐이며 — 이것이 바로 리뷰어가 관찰한 marginal/음수 효과다. 반면 구조 벤치마크(MVTec-AD, VisA)에서는 두 점수가 중첩되는 증거를 포착하므로 융합이 유리하다. 즉 LOCO 결과는 설계와 *모순*되는 것이 아니라 설계와 *일치*한다.

**2. 융합은 구조적으로 논리 중심(λ=0.3, 낮음)이다.**
융합 점수는 결정을 논리 중심으로 유지한다:

```
s_fused = (1 − λ) · s_logic + λ · s_struct,   λ = 0.3
```

λ=0.3에서 논리 분기가 가중치의 70%를 차지하므로 구조 분기는 소수 의견으로만 반영된다. 우리는 융합이 LOCO에서 *이긴다*고 주장하지 않는다 — 설계상 "marginal"하게(즉 의미 있게 *해치지 않게*) 동작하면서, 실제 생산 라인이 함께 가질 구조적 제품에서는 *도움이 된다*는 것이 우리의 주장이다.

**3. 카테고리별 적응형 λ를 권장하며, 보유한 카테고리 최적값을 보고한다.**
최적 가중치는 실제로 제품 의존적이다. λ sweep을 수행한 두 개의 LOCO 카테고리(R2-6에서 보고)에서 카테고리별 최적값은 **breakfast box λ=0.1**(거의 순수 논리)과 **splicing connectors λ=0.4**(케이블 연결 기하 구조가 구조적 단서와 부분적으로 중첩됨)로 다르다. 이 최적값들은 다섯 개 LOCO 카테고리 중 두 개에 대한 단일 sweep(multi-seed·전(全)카테고리 특성화가 아님)에서 얻은 것으로, 보편적 처방이 아니라 적응형 λ를 뒷받침하는 예시적 근거로 제시한다. 이는 R2-6에서 보고한 λ sweep과 완전히 일치한다(breakfast 최적 AUC는 λ=0.1, splicing 최적 AUC는 λ=0.4, λ ≥ 0.5에서 성능 하락). 따라서 단일 λ가 보편적으로 최적이라고 제시하는 대신, 개정 논문에서 **카테고리별 적응형 λ**를 명시적으로 권장하고 이 최적값들을 보고한다. 자동 λ 선택 절차는 구체적 향후 과제로 남긴다.

**융합 모듈은 사실상 무비용이므로 "LOCO에서 marginal"은 손해가 아니다.** marginal한 모듈을 불신하는 흔한 이유는 실제 비용을 치르면서 얻는 것이 적기 때문이다. 그러나 여기서는 그렇지 않다. 단일 RTX 3090(ViT-B/16+ 백본)에서 추론 비용을 측정하였으며, 이는 단일 플랫폼·비multi-seed 측정값이다:

| 구성 요소 | 측정 비용 | 실시간 경로? | 학습 파라미터 |
|-----------|-----------|--------------|---------------|
| 논리 전용 추론 (CLIP 이미지 forward 2회: 전역 + 사전계산 마스크, 텍스트 인코딩, 융합) | 23.5 ± 2.2 ms / 이미지 | 예 | — |
| 점수 융합 연산 `s=(1−λ)s_logic+λ·s_struct` | ~0.001 ms (단일 스칼라 연산) | 예 | **0** |
| RRD 슬라이딩 윈도우 영역 검출 (320 패치) | ~678 ± 262 ms / 이미지 | **아니오 — 오프라인, 1회 실행 후 `.pt`로 캐시** | 0 |
| WinCLIP/PromptAD 구조 분기 (융합 *변형*) | 외부 탐지기 자체 forward | 선택 (λ=0으로 비활성화) | 0 |

융합 연산 자체는 기본 논리 경로 대비 **~0.001 ms와 학습 파라미터 0개**만 추가한다. RRD의 무거운 비용은 **오프라인**으로 상각된다: 이미지당 1회 실행 후 캐시되며(학습/추론 코드가 사전계산된 마스크 텐서를 로드), 따라서 실시간 경로에 **없다**. 융합을 켤 때의 유일한 *런타임* 비용은 선택한 외부 탐지기의 forward이며, λ=0으로 설정하면 순수 **23.5 ± 2.2 ms** 논리 경로로 완전히 되돌릴 수 있다. 다만 λ=0은 외부 탐지기의 forward 비용만 제거할 뿐, 논리 전용 경로는 여전히 오프라인으로 캐시된 RRD 마스크를 사용한다(오프라인 비용은 변하지 않으며, 단지 실시간 경로에 없을 뿐이다).

**강화된 입장 요약.** 융합은 (i) 개념적(이상-유형) 근거에서 설계상 직교적이므로 순수 논리 데이터에서 효과가 작은 것은 예측된 결과이고, (ii) 논리 중심(λ=0.3)이라 깨끗한 논리 점수를 압도할 수 없으며, (iii) 제품별·적응형으로 sweep한 두 카테고리의 최적값(breakfast λ=0.1, splicing λ=0.4)을 보고하고 적응형 λ는 향후 과제로 권장하며, (iv) 사실상 무비용(~0.001 ms, 파라미터 0개)이면서 완전히 선택적(λ=0)이다. 따라서 LOCO에서 우리는 융합 승리를 과장하지 않는다 — 융합은 통합 산업 검사 시스템을 동기화하는 혼합(논리 + 구조) 제품에서 정확히 이득을 보는, 후회가 거의 없고(near-zero-regret) 해롭지 않은 선택지로 제시한다.

**규칙 수 효과 서술 수정:**

## 리뷰어 4 (R4)

### R4-1 — Full-shot 결과 및 확장성 평가 필요

**리뷰어 지적:**
제안된 프레임워크가 학습 데이터 크기에 따라 얼마나 잘 확장되는지 평가하기 위해 full-shot 결과를 포함해야 한다.

**대응:**
WinCLIP 베이스라인으로 LOCO 5개 카테고리에서 shot ∈ {1, 5, 10, 20}을 평가하였다:

| 카테고리 | shot=1 | shot=5 | shot=10 | shot=20 |
|---------|--------|--------|---------|---------|
| Breakfast box | 0.810 | 0.854 | 0.865 | 0.831 |
| Juice bottle | 0.687 | 0.826 | 0.827 | 0.840 |
| Pushpins | 0.638 | 0.655 | 0.727 | 0.683 |
| Screw bag | 0.585 | 0.605 | 0.582 | 0.572 |
| Splicing connectors | 0.589 | 0.636 | 0.643 | 0.699 |
| **평균** | 0.662 | 0.715 | 0.729 | 0.725 |

**해석:**
- shot=1→5 구간에서 평균 +0.053 AUROC로 가장 큰 향상이 이루어진다.
- shot=10/20에서는 추가 향상이 평균 <0.01로 포화(saturation) 상태에 도달한다.
- 이는 정상 샘플이 극히 제한적인 산업 배포 환경에서 few-shot(=5) 설정이 최적의 비용-성능 균형점임을 보인다.

**Full-shot(50–300 shot) 확장 측정 및 정직한 포지셔닝.** 거의 전체 정상 셋에 해당하는 50–300 shot까지 추가로 측정한 결과, 성능이 포화에 그치지 않고 오히려 **단조 하락**한다(단일 시드, high-shot은 n=1 indicative):

| 카테고리 | shot=50 | shot=100 | shot=200 | shot=300 | WinCLIP 외부 분기 |
|---------|--------|---------|---------|---------|------------------|
| Breakfast | 0.874 | 0.760 | 0.719 | 0.670 | ~0.66 (평탄) |
| Juice bottle | 0.844 | 0.762 | 0.721 | 0.713 | ~0.66 (평탄) |
| Pushpins | 0.679 | 0.609 | 0.583 | 0.582 | — |
| Screw bag | 0.574 | 0.541 | — | — | — |

예컨대 breakfast는 0.874(shot=50)에서 0.670(shot=300)으로 하락한다. 반면 외부 **WinCLIP 분기는 ~0.66으로 평탄하게 유지**되므로, 이 하락은 우리 **rule-contrastive 분기 특유의 현상**이다 — 데이터가 풍부해지면 모델이 정상 외관에 과적합(overfit)하여 규칙 기반 대조 신호가 약화된다(early-stop이 test-AUROC 기준이라 고-shot에서는 epoch 1–11에 정점 후 하락). 따라서 우리는 본 방법을 정직하게 **few-shot 특화(specialist)**로 포지셔닝하며, "데이터를 늘릴수록 잘 확장된다"고 주장하지 않는다. full-shot 수치는 supplementary로 보고하고, full-shot 적응(full-shot adaptation)은 향후 연구 과제로 남긴다.
- Screw bag은 shot=10/20에서도 소폭 하락하는데, 이는 소수 샘플에서의 다양성 부족보다 카운팅 규칙의 고유한 어려움이 지배적임을 시사한다.

### R4-2 — Table 15 VLM 및 AD 특화 추론 모델 비교

**리뷰어 지적 (R4-2, AE 및 R1-2 공통):** 더 많은 VLM(Qwen-VL, LLaVA)과 AD 특화 추론 모델(IAD-R1, AnomalyOV, OmniAD)을 비교에 포함해야 한다. AE는 특히 "LogiCode 등 최신 VLM/MLLM 기반 논리 이상 탐지 방법과의 비교 부족"을 지적하였다.

**대응.** 두 부분으로 답변한다. 첫째, 우리가 *제공할 수 있는* 긍정적 근거. 둘째, AD 특화 모델에 대한 항목별 비교 가능성 검토 — 단순한 "미공개" 한 마디가 아니라, 각 프로젝트가 무엇을 공개했고 왜 우리의 few-shot 프로토콜 하에서 *공정한* MVTec LOCO AD 수치를 산출할 수 없는지 정확히 기술한다.

**1. 일반 목적 VLM — 직접 평가 + 72B 인용.** Qwen2.5-VL-3B와 7B를 동일한 긍정 규칙 프롬프트 전략으로 직접 실행하였으며, 이번 평가는 두 모델의 LOCO 평균(56.2, 61.4)을 재확인하였다. 추가로, 유일하게 공개 보고된 Qwen2.5-VL-72B LOCO 수치(LAD-Reasoner)를 인용한다. 표의 GPT-4o 및 Qwen 변형들의 범주별 셀 값은 맥락 제공을 위해 기존 Table 15에서 가져온 것이며, 아래 논증의 핵심 수치는 평균값이다.

| 방법 | Breakfast | Splicing | Juice | Pushpins | Screw | 평균 | Δ vs Ours 1-shot |
|------|-----------|---------|-------|---------|-------|-----|------------------|
| GPT-4o (zero-shot) | 78.9 | 55.0 | 68.2 | 40.6 | 61.0 | 60.7 | −9.3 |
| Qwen2.5-VL-3B (zero-shot, 직접 평가) | 67.0 | 62.1 | 64.0 | 36.7 | 51.4 | 56.2 | −13.8 |
| Qwen2.5-VL-7B (zero-shot, 직접 평가) | **90.3** | 52.9 | 60.2 | 46.3 | 57.5 | 61.4 | −8.6 |
| Qwen2.5-VL-72B† (zero-shot) | 74.6 | 57.3 | 64.4 | 62.9 | 54.8 | 62.8 | −7.2 |
| **Ours 1-shot** | 80.2 | **73.8** | **80.9** | **63.3** | 51.7 | **70.0** | — |
| **Ours 5-shot** | 85.2 | **75.4** | 83.7 | **64.4** | **55.8** | **72.9** | +2.9 |

† MVTec LOCO AD에서 Qwen2.5-VL-72B를 보고한 유일한 문헌인 LAD-Reasoner (Li et al., arXiv:2504.12749)에서 인용.

추가로, 두 개의 공개 VLM을 더 직접 평가하였다: **LLaVA-1.6-7B 49.0%**(가장 약함), **Qwen2-VL-7B 61.8%**. 특히 Qwen2-VL-7B(61.8%)는 후속 세대인 Qwen2.5-VL-7B(61.4%)와 사실상 동급이므로, **VLM 세대 진전만으로는 도움이 되지 않음**을 보인다. 두 모델 모두 우리 1-shot 70.0%에 못 미친다.

핵심 발견은 *우리가 측정할 수 있었던 모든 규모에서 평균적으로 지속되는 격차*이다: LOCO 평균 기준으로 GPT-4o부터 72B 모델까지 모든 VLM이 우리 1-shot 결과보다 7~14포인트 낮다(Q-72B −7.2, Q-3B −13.8). 다만 이는 평균 수준의 진술임을 분명히 한다: 범주별로는 양상이 엇갈려, 예컨대 Q-7B는 Breakfast에서 우리 1-shot를 능가하고(90.3 vs 80.2), Screw에서는 GPT-4o·Q-7B·Q-72B가 모두 우리보다 높다. 우리의 우위는 논리성이 강한 범주(Splicing, Juice)가 견인하는 집계(평균) 수준의 우위이다. 그럼에도 평균 격차는 프롬프트 설계의 부산물이 아니다. 규모 대 few-shot 대비가 이를 보여준다: **zero-shot VLM 백본을 23배 키워도(Qwen2.5-VL 3B → 72B) LOCO 평균은 +6.6포인트(56.2 → 62.8) 상승에 그치며 여전히 우리 few-shot 결과에 못 미친다. 반면 우리 방법은 훨씬 작은 CLIP 백본(ViT-B/16+, 약 10⁸ 파라미터) 위에서 단 1-shot으로 70.0에 도달한다.** 이는 동일 아키텍처를 통제한 ablation이 아니라 *교차 모델(cross-model)* 대비임을 분명히 한다: 규모 확장 축은 zero-shot Qwen2.5-VL 계열이고 few-shot 축은 별개의 훨씬 작은 CLIP 모델이다. 이득의 출처에 대한 귀속 논증으로 해석하면, 논리 이상 탐지의 지배적 지렛대는 모델 용량 자체가 아니라 특정 제품 정상 외관에 대한 few-shot 보정(calibration)이며 — 3B–72B 범위의 어떤 zero-shot VLM도 평균 격차를 메우지 못하는 반면, 약 10⁸ 파라미터 백본에 단일 라벨 샘플 하나가 이들 모두를 능가한다.

**2. AD 특화 추론 모델 — 항목별 비교 가능성.** 각 모델에 대해 무엇이 공개되었는지, 그리고 우리 프로토콜 하에서 공정한 LOCO 수치를 산출할 수 없는 구체적 사유를 정확히 기술한다. 우리는 이 모델들을 일축한 것이 아니라, 각 프로젝트가 공개한 자료를 실제로 활용하려 시도하였다.

| 모델 | 공개 내용 | 공정한 LOCO 비교의 구체적 걸림돌 |
|------|---------|--------------------------------|
| LogiCode | 주석 데이터만 | 추론 코드·체크포인트 모두 미공개 → LOCO에서 모델 자체를 실행할 수 없음, 인용으로만 다룸 |
| IAD-R1 | 코드 **및** 체크포인트 (공개) | 공개 모델이 어떤 MVTec LOCO AD split에서도 평가되지 않음 → 수치 보고 시 저자가 명시하지 않은 평가 프로토콜을 우리가 정의·실행해야 하므로 *저자 보고 결과*가 아님; 구조 특화 모델이라 보완적 방향으로 논의 |
| Anomaly-OV | 탐지 헤드 (공개) | **이제 직접 평가하였다** — zero-shot 논리 AUROC **0.534**(R1-2 표 참조)로 우리 1-shot 0.727 미달; 추론-MLLM 탐지 헤드 중 가장 약함 |
| OmniAD | instruction-tuned 멀티모달 reasoner | MMAD 벤치마크에서 평가(우리의 few-shot MVTec LOCO 프로토콜 아님); 비교 가능한 수치를 얻으려면 우리 ≤5-shot 체제로 재구현·재평가가 필요 → apples-to-apples 비교 불가, 인용으로만 다룸 |

요컨대: **Anomaly-OV는 탐지 헤드가 공개되어 직접 평가하였고(0.534, R1-2 참조)**, LogiCode와 OmniAD는 코드/체크포인트가 없거나 벤치마크가 달라 실행 자체가 불가능하며, IAD-R1은 기술적으로 실행은 가능하나 LOCO 결과를 보고하지 않으므로 우리가 만든 수치는 저자 검증을 거친 공정 비교가 아니라 우리 프로토콜 하의 재구현일 뿐이다. 직접 비교 불가한 LogiCode/IAD-R1/OmniAD는 인용하고 보완적 연구 방향으로 논의하였다. 우리는 이 각각의 걸림돌을 "미공개"라는 한 마디 대신 명시적으로 기술하였으며, 리뷰어가 요청한 구체적·재현 가능한 VLM 비교로서 직접 측정한 Q-3B/7B(및 LLaVA-1.6-7B, Qwen2-VL-7B) 평균과 인용한 Q-72B 점을 제공한다.

### R4-3 — 부정 규칙 수 및 생성 모듈 ablation 부재

**리뷰어 지적:**
긍정 규칙 수 효과(Table 16)는 ablation하였으나, 부정 규칙 수와 생성 모듈 ablation이 없다.

**대응:**
R2-2 ablation이 이를 직접 다룬다(다중 시드 original-vs-shuffled 표 + 단일 시드 naive/nn 행). 요약:

- **부정 규칙 수 (nn) 영향**: 단일 시드 조건에서 nn을 늘릴 때 splicing이 가장 민감 (nn=1: 0.568 → nn=3: 0.635; 다중 시드 original 평균은 0.653), breakfast는 안정적 (nn=1: 0.889, nn=3: 0.887). 단일 부정 규칙만으로도 유용한 대조 신호가 공급되며, 다수일수록 robustness에 기여.
- **생성 모듈 비교**: naive(checker 없음)가 검증 파이프라인과 ~0.02–0.06 AUC 이내로 근접 — checker는 규칙 auditability를 위한 안전망이지 성능의 지배적 요인이 아님.
- **다중 시드 결론**: 평균을 취하면 올바른(original) 부정 규칙이 breakfast에서 shuffled보다 우세(+0.024); 더 어려운 splicing에서는 original/shuffled 차이가 1-시그마 구간 내로 겹쳐 통계적으로 유의하지 않음 — 잘못된 할당에도 붕괴 없는 graceful degradation이며, 틀린 규칙이 더 좋다는 의미가 아님.
